System Requirements

Most problems users experience are caused by out of date versions of Java.

System Demonstration

Introduction

This website is intended to assist in building intuition about the dynamics of two-player games. It provides visualizations for the best-response dynamics, as well as for two population discrete-time and continuous-time replicator dynamics.

This is a tutorial intended to show you what the system can do and how to use it. Opening another browser window or tab and following along is highly encouraged. Please see the system requirements page as well.

Basic Controls and Adding Games

To begin, you are presented only with some controls. At the outset, we are most concerned with the "add game" button.

Clicking the "add game" button will reveal an interface for entering game information. By default, you are free to enter all parameters for the game. However, there is a drop-down available with which you can select to use one of several common games instead of entering your own.

Selecting "Stag Hunt" from the drop-down will enter values for the game parameters automatically. We can also give the game a label to more easily remember what game it is later. Giving the game a label is optional. If one is not provided, a (rather ugly) one will be generated.

You can re-select drop-down entries and change the game parameters as much as you would like. When you are satisfied with the parameter values, clicking the "save game" button will save the parameters and show you the normal (strategic) form of the game.

Generating Visualizations

Once we have the normal (strategic) form of the game, clicking the "generate graphs" button will (perhaps a bit slowly) reveal four graphs. If the graphs are not generated, please check that your system meets the system requirements.

The top-left graph shows the best-response dynamics for the game. The axes in each graph correspond to the cells of the normal form. For example, the top-left corner of the graph is the point where the row player always chooses the upper row and the column player chooses the left column.

The bottom-left graph shows a differential vector field according to the replicator dynamics. The upper-right graph shows trajectories of the discrete-time replicator dynamics, and the bottom-right graph shows trajectories of the continuous-time replicator dynamics.

Editing Games

At any time, you can go back and edit the game you have entered. Clicking the "edit game" button will return you to the game editing interface. Changing the drop-down selector to "Free Entry" will let us change the values of the game parameters to anything we would like. For example, we might enter these payoff values.

When we are done editing the parameters, we can again click the "save game" button and return to viewing the normal form of the game. Clicking "generate graphs" with these new parameters will once again reveal several graphs visualizing the various aspects of the dynamical systems mentioned earlier.

Using Multiple Games at Once

Sometimes it is useful to compare games directly. While we could look at one game's dynamics, edit the parameters, and then look at another's, it is often easier to have both sets of graphs on the screen at the same time. This system makes that relatively easy.

At any time, you can again click the "add game" button, which will insert an entry form for another game at the bottom of the page. This form works exactly like the the first one we demonstrated earlier.

Having many games saved at the same time can be unwieldy. Therefore, it is possible to "collapse" the games, leaving only their label showing. This is accomplished either by clicking the "collapse all" button, or by clicking on an individual game's label. To re-expand the game, click on the label a second time, or click the "expand all" button.

Finally, games can also be sorted in the list by dragging them up and down. A blue area will show where the game will be inserted in the list once it is released.
